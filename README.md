# POI Inference on UK Bicycle Usage

This repository contains the codebase and documentation for researching the spatial relationship between Points of Interest (POIs) and bicycle-sharing behaviors in the UK. The project is structured to facilitate transparent data extraction, rigorous preprocessing, and robust spatial modeling, making it fully reproducible for research purposes.

## Repository Structure

The project directory is organized into several key folders to separate the data pipeline, analysis, and modeling components:

*   **`data/`**: Intended for storing the raw and processed datasets and boundary mesh file generated by the pipeline.
*   **`data_extraction_and_preprocessing/`**: Contains all Python and R scripts required to extract, clean, and format the geographic and bikeshare data.
*   **`eda/`**: Dedicated to Exploratory Data Analysis notebooks and scripts.
*   **`model/`**: Contains the core statistical modeling scripts and evaluation metrics.
*   **`outputs/`**: Contains output tables, graphs and maps.

## Data Extraction and Preprocessing Workflow

To replicate the research and generate the required datasets, scripts within the `data_extraction_and_preprocessing` directory must be executed in a specific sequential order. 

Follow these steps to establish the dataset:

1.  **Execute Initial Data Creation**: 
    *   Run `main.py` to initiate the pipeline and create the foundational datasets (e.g., `jan_feb_winter_dataset.csv`, POI distances, etc.).
2.  **Filter Geographic Stations**: 
    *   Run the R script `Selecting_100_closest_stations.R` to isolate the relevant geographical subset (100 closest stations) for the study.
3.  **Format Seasonal Data**: 
    *   Run the R script `Prepare_Seasonal_datasets.R` to partition and format the data according to the four meteorological seasons.
4.  **Establish Spatial Boundaries**: 
    *   Run the R script `create_boundary_mesh.R` to construct the spatial mesh required for the downstream spatial models.

### Note
To replicate the results use the given boundary_mesh.rds in the `data` directory (without running the create_boundary_mesh.R).
Completing this sequence ensures all dependencies and seasonal data subsets are properly extracted and staged for statistical modeling.

## Modeling and Evaluation

All predictive and inferential models are housed in the `model` directory. This section of the repository contains R scripts specifically dedicated to:

*   **Spatial Data Models (SDM)**: Utilizing Template Model Builder (TMB) frameworks to handle complex spatial autocorrelation and POI inference.
*   **Seasonal Evaluations**: Dedicated evaluation workflows assessing model performance independently across all four seasons to account for temporal variations in bicycle usage.

Run the model R scripts in the `model` directory. The model outputs will be saved in the `outputs` directory.
